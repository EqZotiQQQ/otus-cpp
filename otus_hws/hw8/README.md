# Использование
```
./duplicate_finder --include <paths...> [options]
```

# Пример:
```
./duplicate_finder \
  --include otus_hws/hw8/assets \
  --depth 42 \
  --min-size 1 \
  --block-size 3 \
  --i ".txt" ".log" \ 
  --hash md5 \
  --exclude exclude_dir
```


# Подробное описание

--help

--include <paths...>
Список директорий, где будет выполняться поиск.

--exclude <paths...>
Пути, которые нужно исключить из поиска.

--depth <N>
Максимальная глубина рекурсивного обхода директорий.
0 — только текущий уровень, 1 — +1 уровень вложенности и т.д.

--min-size <bytes>
Минимальный размер проверяемых файлов

--block-size <bytes>
Размер блока для поблочного чтения и хеширования.
По умолчанию: 4096. (В тестовой подборке нет таких файлов, так что лучше прокинуть 3 байта, так хоть какая-то имитация работы будет)

--hash <name>
Алгоритм хеширования. Только:
md5

--i <masks...>
Список масок/расширений файлов (без учёта регистра), которые нужно включить в анализ.
Например:
--i ".txt" ".log"
